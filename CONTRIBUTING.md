# Contributing

## 基本フロー

```text
Issue → mainをPull → Branch → 編集 → Stage → Commit → Push → Pull Request → Review → Merge
```

## Branch名

```text
feature/<内容>
fix/<内容>
docs/<内容>
experiment/<内容>
analysis/<内容>
```

## Commit

- 1つの目的に対応する変更をまとめる
- 後から内容が分かるメッセージを付ける
- 関係のない整形やファイル追加を混ぜない

## Pull Request

- 変更内容
- 確認方法
- 結果
- 関連Issue

を記載してください。
